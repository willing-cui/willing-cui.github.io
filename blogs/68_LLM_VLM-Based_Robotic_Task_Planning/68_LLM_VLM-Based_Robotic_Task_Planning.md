
## 一、 引言：从“文本规划”到“物理控制”的范式迁移

传统机器人任务规划遵循“感知 → 符号推理 → 技能调用”的模块化流水线（如 SayCan 方案）。这种模式存在**语义鸿沟**（Semantic Gap）：LLM 生成的文本计划（如 `"pick(apple)"`）往往缺乏对物理可行性的理解。

**VLA（Vision-Language-Action）模型** 的出现标志着范式的根本转变：
- **核心命题**：将机器人**连续控制动作**视为一种特殊的“语言”，利用 LLM 的序列建模能力，实现**端到端**的“图像+指令 → 动作”映射。
- **技术标志**：**RT-2**（2023）与 **OpenVLA**（2024）是这一领域的两个里程碑，前者验证了 VLA 的泛化极限，后者构建了开源社区的基座。

## 二、 技术基石：动作 Token 化（Action Tokenization）原理

VLA 模型的核心突破在于解决了“如何用离散 Token 表示连续物理动作”的问题。

### 1. 数学原理：连续空间的离散量化
设机器人动作空间为 $a \in \mathbb{R}^d$（如末端位姿、关节角度）。**动作 Token 化** 通过量化函数 $Q$ 将其映射到离散整数集：
$$
\text{Token}\_i = Q(a\_i) = \left\lfloor \frac{a\_i - \min\_i}{\max\_i - \min\_i} \times (K-1) \right\rfloor
$$
其中 $K$ 为量化等级（通常 $K=256$），$\text{Token}\_i$ 对应词汇表中的特殊 Token（如 `<action_42>`）。

### 2. RT-2 的具体实现
- **动作维度**：8 维 = 6D 末端位移（$\Delta x, \Delta y, \Delta z, \Delta \text{roll}, \Delta \text{pitch}, \Delta \text{yaw}$） + 夹爪开合 + 终止标志。
- **量化方案**：每维均匀量化为 256 个 Bins，共需 8 个 Token 表示一个完整动作步。
- **词汇表扩展**：在 PaLM-E/PaLI-X 的词汇表中预留 256 个未使用的 Token 专门表示动作 Bins。

### 3. OpenVLA 的改进
- **维度调整**：输出 7 维动作（通常为 $\Delta$ 位姿），每维同样离散化为 256 个区间。
- **非均匀量化**：部分实现采用基于数据分位数的**非均匀量化**，以更好适应真实动作分布的边界。

## 三、 架构设计：多模态融合与推理引擎

VLA 普遍采用“视觉编码器 → 投影器 → LLM 主干 → 动作头”的通用架构，但 RT-2 与 OpenVLA 在组件选择上存在显著差异。

### 1. 视觉编码器（Vision Encoder）对比

| 模型 | 视觉编码方案 | 技术细节与原理 |
| :--- | :--- | :--- |
| **RT-2** | **单路 ViT** | 使用 PaLI-X 或 PaLM-E 自带的 ViT，依赖超大模型容量（55B）隐式融合语义与几何特征。 |
| **OpenVLA** | **双路融合** (DINOv2 + SigLIP) | **显式解耦**：<br>• **DINOv2**：自监督学习，擅长**空间定位**与几何特征。<br>• **SigLIP**：对比学习，擅长**语义理解**。<br>• 输出特征在通道维度拼接，通过 MLP 投影对齐。 |

> **双路编码优势**：即使模型较小（7B），也能通过架构先验获得极强的空间感知，避免语义干扰几何，这是 OpenVLA 高性能的关键。

### 2. 主干网络（Backbone）与投影器

- **RT-2**：基于 **PaLM-E (55B)** 或 **PaLI-X**。这些是 Google 内部为多模态特化的巨型模型，其训练数据（WebLI）与机器人数据在隐空间深度对齐。
- **OpenVLA**：基于 **Llama 2 (7B)**。选用流行开源底座，证明“通用 LLM + 适配器”路线的可行性。投影器通常为 2 层 MLP，负责将视觉特征投影到 Llama 2 的文本嵌入空间。

### 3. 动作头（Action Head）
两者均采用**轻量化设计**，仅使用线性层将 LLM 的隐状态 $h\_t$ 映射到动作 Token 的 Logits：
$$
\hat{a}\_t = \text{Softmax}(W\_a h\_t + b\_a)
$$
这利用了 LLM 强大的表征能力，动作头仅负责最终的“映射”任务。

## 四、 训练策略：知识迁移与数据混合

### 1. RT-2 的混合微调（Co-finetuning）
- **动机**：防止模型在学动作时“忘记”互联网知识（灾难性遗忘）。
- **数据混合**：在微调阶段，**交替**使用机器人轨迹数据（图像+动作）和互联网图文数据（VQA、Captioning）。
- **损失函数**：
  $$
  \mathcal{L} = \lambda\_{\text{robot}} \mathcal{L}\_{\text{robot}} + \lambda\_{\text{vlm}} \mathcal{L}\_{\text{vlm}}
  $$
  其中 $\mathcal{L}\_{\text{robot}}$ 是动作预测的交叉熵损失，$\mathcal{L}\_{\text{vlm}}$ 是视觉语言任务的损失。

### 2. OpenVLA 的大规模真实数据预训练
- **数据源**：基于 **Open X-Embodiment (OXE)** 数据集，包含约 97 万条来自多个机器人平台的真实演示。
- **策略**：直接在多样化机器人数据上微调预训练的 Prismatic VLM（基于 Llama 2 + 双路视觉）。
- **优势**：构建对**物理交互本身**具有强先验的通用策略，而非仅仅依赖 VLM 的语义先验。

## 五、 任务规划机制：隐式推理与显式 CoT

### 1. 隐式规划（Implicit Planning）
这是 VLA 的主流模式。**规划与执行是耦合的**，模型不输出文本步骤，而是直接输出动作序列。
- **原理**：模型在生成动作 Token 时，内部隐状态 $h\_t$ 已经编码了**任务进度和子目标**。例如，当模型输出“移动至香蕉”的动作时，它已经推理出“下一步需要抓取”。
- **数学本质**：学习策略 $\pi$，使得 $\pi(o\_t, s) = \arg\max P(a\_t | o\_{1:t}, s, a\_{1:t-1})$，其中 $o\_t$ 是观测，$s$ 是指令。

### 2. 显式思维链（Explicit Chain-of-Thought）
通过 Prompt 工程让模型先输出“思考”文本，再输出动作：

指令: 把香蕉放进碗里
模型输出: Think: 我需要先找到香蕉，然后抓取它，再移动到碗上方，最后松开夹爪。 <eos> <action\_123>...

- **作用**：提高规划的可解释性，适合复杂多步任务。RT-2 论文中展示了这种能力。

## 六、 RT-2 与 OpenVLA 的深度对比

| 维度 | **RT-2 (Google DeepMind)** | **OpenVLA (Stanford/Toyota)** |
| :--- | :--- | :--- |
| **根本定位** | **闭源技术标杆**，验证 VLA 的极限泛化 | **开源基础设施**，推动社区研发与微调 |
| **模型规模** | 55B / 5B（巨型，依赖容量） | 7B（紧凑，依赖架构与数据） |
| **视觉编码** | 单路 ViT（端到端隐式融合） | **双路融合**（DINOv2 + SigLIP，显式解耦） |
| **训练数据** | 混合互联网图文 + 机器人数据 | 大规模真实机器人数据（OXE） |
| **泛化机制** | **语义泛化**（利用 Web 知识理解新概念） | **物理泛化**（跨本体、跨场景的鲁棒执行） |
| **性能表现** | 基准水平 | **成功率高出 RT-2 约 16.5%**（论文数据） |
| **可访问性** | 黑盒 API，不可微调 | 完全开源（权重/代码），支持 LoRA/Full FT |

> **反直觉洞察**：OpenVLA 以 1/8 的参数量，在多项基准上超越了 RT-2。这证明了在机器人领域，**数据的多样性（OXE）和架构的针对性（双路编码）** 比单纯的“大模型”更重要。

## 七、 局限性与未来方向

1.  **动作精度瓶颈**：离散化（256 Bins）会损失控制精度，难以处理高精度装配或高速动态任务。
2.  **长程规划模糊**：对于“做一顿饭”级的长任务，端到端模型易遗忘最终目标，需结合高层 LLM 进行显式分层规划（如 VLA-OS 框架）。
3.  **数据稀缺**：高质量、大规模的机器人演示数据依然难以获取，限制了泛化上限。
4.  **安全与验证**：如何形式化验证 VLA 生成动作的安全性，仍是开放难题。

**未来趋势**：**分层 VLA**（高层 LLM 规划 + 底层 VLA 执行）、**3D 视觉注入**（点云融合）、**扩散动作**（生成更平滑的动作分布）。

## 八、 总结

**RT-2** 与 **OpenVLA** 共同确立了 VLA 作为机器人任务规划新范式的地位。它们通过 **“动作 Token 化”** 技术，将 LLM 的序列预测能力直接应用于物理控制。RT-2 代表了 **“VLM 向下兼容”** 的工业级路线，而 OpenVLA 代表了 **“机器人数据向上融合”** 的开源路线。未来，结合分层规划与更丰富感知模态的 VLA 模型，将是实现通用具身智能的关键路径。
